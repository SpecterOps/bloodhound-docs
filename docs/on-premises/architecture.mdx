BloodHound Enterprise on-premises gives you full control over your deployment infrastructure while maintaining the same powerful identity security capabilities as the SaaS version.

## Deployment architecture

BloodHound Enterprise on-premises consists of two primary parts:

- **BloodHound Enterprise host (Linux)** - Runs the BloodHound application, database, and supporting infrastructure
- **Collector hosts (Windows)** - Run lightweight collector services (SharpHound and/or AzureHound)

### Core components

The BloodHound Enterprise host includes the following components:

| Component | Purpose |
|-----------|---------|
| **BloodHound Enterprise API** | Application server, UI, graph analysis, and collector ingestion |
| **PostgreSQL 18.x** | Unified database for application data and graph storage |
| **Kubernetes (k0s)** | Container orchestration platform (embedded cluster only) |
| **BloodHound Enterprise Portal** | Deployment management, configuration, and updates |

### Data collectors

Collectors run separately from the BloodHound Enterprise host and gather configuration data from your identity infrastructure:

| Collector | Target Environment | Data Collected |
|-----------|-------------------|----------------|
| **SharpHound Enterprise** | Active Directory | AD objects, relationships, ACLs, sessions |
| **AzureHound Enterprise** | Azure / Entra ID | Azure AD objects, role assignments, resource relationships |
| **OpenGraph** | Custom sources | User-defined graph data (JSON) |

## Data flow

Data flows through the system in the following sequence:

1. **Collection** - Collectors gather configuration data from AD, Azure, or custom sources
2. **Transmission** - Data is transmitted over encrypted HTTPS/TLS 1.3 to the BloodHound Enterprise API
3. **Processing** - The BloodHound Enterprise API processes and stores data in PostgreSQL
4. **Analysis** - Graph analysis identifies privilege relationships and attack paths
5. **Visualization** - Results are displayed in the BloodHound UI

<Note>
  Collectors have zero local storage of collected data. All data is transmitted directly to the BloodHound Enterprise host and stored in PostgreSQL.
</Note>
