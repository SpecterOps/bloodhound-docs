## Configure BloodHound

Ensure you have configured an IDP for BloodHound as described in **Order of Operations** before proceeding.

<Steps>
  <Step title="Open the SSO Configuration page">

  <Note>You must be logged in as an Administrator to perform this action.</Note>

  In the left menu, click **Administration** > **Authentication** > **SSO Configuration**.
  </Step>

  <Step title="Select a provider type">

  Click **Create Provider** > **{auth_mode} Provider**.

      <img
        src="/images/sso/create_provider.png"
        alt="A view of the SSO Configuration page with the Create Provider button highlighted."
      />
  </Step>
  <Step title="Enter provider details">

  Enter the **{auth_mode}** provider details.

  | Field | Auth Type | Description |
  |-------|-----------|-------------|
  | **Provider Name** | OIDC and SAML | **App integration name** obtained from your [Okta application](/manage-bloodhound/auth/oidc#configure-okta) (or other IDP) |
  | **Client ID** | OIDC only | Obtained from your [Okta application](/manage-bloodhound/auth/oidc#configure-okta) _Client Credentials_ |
  | **Issuer** | OIDC only | URL obtained from your [Okta application](/manage-bloodhound/auth/oidc#configure-okta) _Authorization Server_ |
  | **Metadata File** | SAML only | Metadata XML file you [created previously](/manage-bloodhound/auth/saml-okta#okta-attribute-statements) for Okta (or other IDP) |
  | **Default Role** | OIDC and SAML | Default role assigned to users when no role is provided by the {auth_mode} provider |
  </Step>
  <Step title="Save the configuration">

  Click **Submit**.

  </Step>
</Steps>
