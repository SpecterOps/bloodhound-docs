## Configure Users

By default, all users utilize a username and password via the built-in authentication service. When [creating or modifying a user](/manage-bloodhound/auth/users-and-roles), you can change this setting. 

When creating a new {auth_mode} user, ensure the {auth_mode} user does not share an email address with any other users (across all authentication methods).

1.  Log in to BloodHound as an Administrator.

1.  Open the slider on the left and click **Administration**.

    <img
      src="/images/sso/select_administration.png"
      alt="A view of the BloodHound UI with the Administration option highlighted in the left slider menu."
    />

1.  Under the **Users** section, click **Manage Users**.

    <img
      src="/images/sso/select_manage_users.png"
      alt="A view of the Administration page with the Manage Users option highlighted."
    />

1.  Locate the user you want to configure with authentication, click the hamburger menu button on the right side of the row, then **Update User**.

    <img
      src="/images/sso/user_hamburger.png"
      alt="A view of the Manage Users page with the hamburger menu button highlighted for a user."
    />

1.  In the following dialog, select the **Single Sign-On** authentication method, then select the appropriate SSO provider.

    <img
      src="/images/sso/update_user_no_provision.png"
      alt="A view of the Update User dialog with the Single Sign-On authentication method selected."
    />

    - When Provisioning is enabled without the Modify Role option, a user's role may be updated manually after creation.
    - If both Provisioning and Modify Role are enabled, role updates must come through the SSO provider (manual updates in BloodHound are disabled).

      <img
        src="/images/sso/update_user_with_provision.png"
        alt="A view of the Update User dialog with the Single Sign-On authentication method selected and provisioning enabled."
      />

1.  Click **Save**.
