OpenGraph metadata is optional and is defined at the top level of the payload.

## Supported Metadata Fields

### source_kind

Use `source_kind` to attribute all nodes in a payload file to a source system.

- Type: string
- Scope: applied to all nodes in the same file during ingest
- Example sources: `GitHub`, `Snowflake`, `MSSQL`

BloodHound appends this value to each node's `kinds` array at ingest time.

<Note>
  BloodHound already uses this concept for built-in data sources. Active Directory data uses `Base`, and Azure data uses `AZBase`.
</Note>

## Example

```json
{
  "metadata": {
    "source_kind": "GitHub"
  },
  "graph": {
    "nodes": [],
    "edges": []
  }
}
```

## Troubleshooting

- **`source_kind` does not appear in expected nodes:** Verify that `metadata` is at the top level of the payload and not nested under `graph`.

- **Unexpected kind labels after ingest:** Confirm that your `source_kind` value is intentional because BloodHound appends it to every node in the file.
