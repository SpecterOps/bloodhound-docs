The `metadata` field is used to define system-level behavior for your data payload.

One of its primary uses is registering a `source_kind` in your data payload, which determines how data is included in metrics and analysis.

```json
{
  "metadata": {
    "source_kind": "GitHub"
  }
}
```

A `source_kind` assigns all nodes in a data payload to a source system, such as GitHub. During ingest, BloodHound appends this value to each node's `kinds` array.

BloodHound uses `source_kind` to determine how nodes are included in analysis, metrics, and data management workflows. It is also a key way to distinguish data from different sources in your graph.

<Note>
  BloodHound already uses this concept for built-in data sources. Active Directory data uses `Base`, and Azure data uses `AZBase`.
</Note>

Once registered, it persists and is reused automatically for subsequent uploads that include the associated kinds but do not specify a `source_kind`.

This allows you to set it once and have it apply to all future uploads from the same source system without needing to include it in every payload.

## Metrics

<img
  noZoom
  src="/assets/enterprise-edition-pill-tag.svg"
  alt="Applies to BloodHound Enterprise only"
  style={{ width: "20%" }}
/>

To be included in metrics and analysis (findings), nodes must have a `source_kind` defined at the payload level. 

This is because BloodHound uses `source_kind` to determine which nodes are part of the "source system" for a finding, which in turn determines how exposure and impact metrics are calculated.

The following points describe how `source_kind` influences metrics calculations:

- Exposure metrics
  - Do NOT include the source node itself
  - Only include nodes that can traverse to the source node

- Impact metrics
  - Traverse "downstream" from the target node
  - Stop when another Tier Zero (T0) node is encountered

<Note>
  Nodes beyond a second Tier Zero (T0) node are not included in impact calculations.
</Note>

## Schema

Use the schema below as the source of truth for validation requirements:

```json
{
    "title": "Metadata",
    "description": "Optional metadata about the ingest payload",
    "type": "object",
    "properties": {
        "source_kind": {
            "type": ["string","null"]
        }
    },
    "additionalProperties": false
}
``` 

## Troubleshooting

- **`source_kind` does not appear in expected nodes:** Verify that `metadata` is at the top level of the data payload and not nested under `graph`.

- **Unexpected kind labels after ingest:** Confirm that your `source_kind` value is intentional because BloodHound appends it to every node in the data payload.
