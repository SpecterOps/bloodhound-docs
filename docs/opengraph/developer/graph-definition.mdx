An extension definition schema tells BloodHound how to structure extension-specific [data payloads](/opengraph/developer/graph-data). Use it when you want to produce a <Tooltip tip="A graph that includes structured analysis, pathfinding, and findings based on an installed extension definition schema." cta="Learn more" href="/opengraph/overview#structured-graphs">structured graph</Tooltip> to enable enhanced features and capabilities in BloodHound.

<Tip>
  If you only need <Tooltip tip="A graph that uses only the data payload without structured analysis or pathfinding capabilities." cta="Learn more" href="/opengraph/overview#generic-graphs">generic graph</Tooltip> support, upload a valid OpenGraph data payload and skip this page.
</Tip>

An extension definition schema is a JSON file that OpenGraph extension developers provide for BloodHound users to install on the [OpenGraph Management](/opengraph/extensions/manage) page before they upload data payloads that conform to the schema.

The definitions in the schema are applied to compatible data payloads during ingestion to produce a structured graph with enhanced features and capabilities.

```json
{
  "schema": { ... },
  "node_kinds": [ ... ],
  "relationship_kinds": [ ... ],
  "environments": [ ... ],
  "relationship_findings": [ ... ]
}
```

<ResponseField name="schema" type="object" required>
  Extension metadata, including identity, display name, and namespace.
</ResponseField>
<ResponseField name="node_kinds" type="object[]" required>
  Defines custom node types for extension-specific data payloads and their visual configurations.
</ResponseField>
<ResponseField name="relationship_kinds" type="object[]" required>
  Defines custom edge types for extension-specific data payloads and their traversability behavior.
</ResponseField>
<ResponseField name="environments" type="object[]">
  Defines system-level data sources and principal kinds for extension-specific data payloads.
</ResponseField>
<ResponseField name="relationship_findings" type="object[]">
  Defines findings and remediation guidance that map relationship kinds to analysis output in a target environment.
</ResponseField>

## Naming and namespacing

All defined names must follow namespacing rules:

- Must be prefixed with your namespace
- Must be unique within the extension
- Must include more than just the namespace

<Warning>
  If names are not unique or properly namespaced, the schema upload will fail.
</Warning>

## `schema`

Defines metadata about the extension itself.

```json
{
  "name": "SOOkta",
  "display_name": "Okta Extension (by SpecterOps)",
  "version": "v2.8.1",
  "namespace": "Okta"
}
```

<ResponseField name="name" type="string" required>
  Unique extension identifier used by BloodHound to register the schema.
</ResponseField>
<ResponseField name="display_name" type="string" required>
  Human-readable extension name shown in product experiences.
</ResponseField>
<ResponseField name="version" type="string" required>
  Extension schema version in semantic version format, for example `v1.0.0`.
</ResponseField>
<ResponseField name="namespace" type="string" required>
  Prefix used for all extension-defined kinds and finding names.
</ResponseField>


## `node_kinds`

Defines all node types in your graph.

Each node kind represents an entity type (for example: user, device, or environment).

```json
[
  {
    "name": "Okta_User",
    "display_name": "Okta User",
    "description": "An Okta user account",
    "is_display_kind": true,
    "icon": "user",
    "color": "#d33115"
  }
]
```

<ResponseField name="name" type="string" required>
  Unique, namespaced node kind identifier.
</ResponseField>
<ResponseField name="display_name" type="string" required>
  Human-readable label shown for this node kind in BloodHound.
</ResponseField>
<ResponseField name="description" type="string">
  Optional description that explains what the node kind represents.
</ResponseField>
<ResponseField name="is_display_kind" type="boolean">
  Indicates whether this node kind should be treated as a displayable kind in the product.
</ResponseField>
<ResponseField name="icon" type="string">
  Optional Font Awesome [icon name](https://fontawesome.com/search?s=solid) (without the "fa-" prefix) to show for nodes of this kind in the graph.
</ResponseField>
<ResponseField name="color" type="string">
  Optional Hex color code (in `#RGB` or `#RRGGBB` format, the `#` is required) to apply to nodes of this kind in the graph.
</ResponseField>

## `relationship_kinds`

Defines how nodes can be connected.

```json
[
  {
    "name": "Okta_ResetPassword",
    "description": "Ability to reset passwords or temporary credentials for scoped Okta users",
    "is_traversable": true
  }
]
```

<ResponseField name="name" type="string" required>
  Unique, namespaced relationship kind identifier.
</ResponseField>
<ResponseField name="description" type="string">
  Optional description of what the relationship means.
</ResponseField>
<ResponseField name="is_traversable" type="boolean">
  Controls whether edges of this relationship kind are used in pathfinding and analysis.

  When `is_traversable` is set on a relationship kind, all edges of that kind inherit the same traversability behavior. Only traversable edges are included in pathfinding and considered for findings and analysis.
</ResponseField>

## `environments`

Defines analysis boundaries and data sources.

```json
[
  {
    "environment_kind": "Okta_Organization",
    "source_kind": "Okta",
    "principal_kinds": [
      "Okta_User",
      "Okta_Group",
      "Okta_Application",
      "Okta_ApiServiceIntegration"
    ]
  }
]
```

<ResponseField name="environment_kind" type="string" required>
  Node kind that represents the environment boundary for analysis. Must match a node kind defined in `node_kinds`.
</ResponseField>
<ResponseField name="source_kind" type="string" required>
  Source kind used to associate ingested data with this environment.
</ResponseField>
<ResponseField name="principal_kinds" type="string[]" required>
  Node kinds that BloodHound should treat as principals within the environment.
</ResponseField>

## `relationship_findings`

Defines findings based on relationships.

<Note>
  Extensions that define findings and remediations work in both Community and Enterprise, but findings are visible in Enterprise only.
</Note>

```json
[
  {
    "name": "Okta_Finding_ResetPassword",
    "display_name": "Password Reset Permission Across Privilege Zones",
    "environment_kind": "Okta_Organization",
    "relationship_kind": "Okta_ResetPassword",
    "remediation": {
      "short_description": "Lorem ipsum dolor sit amet, consectetur adipiscing elit.",
      "long_description": "Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.",
      "short_remediation": "Lorem ipsum dolor sit amet, consectetur adipiscing elit.",
      "long_remediation": "Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur."
    }
  }
]
```

<ResponseField name="name" type="string" required>
  Unique, namespaced finding identifier.
</ResponseField>
<ResponseField name="environment_kind" type="string" required>
  Environment kind where this finding should be evaluated.
</ResponseField>
<ResponseField name="relationship_kind" type="string" required>
  Relationship kind that triggers this finding when matching edges are present.
</ResponseField>
<ResponseField name="remediation" type="object">
  Remediation guidance to resolve the finding, including both short and long descriptions.
</ResponseField>
<ResponseField name="remediation.short_description" type="string">
  A concise summary of the recommended remediation action.
</ResponseField>
<ResponseField name="remediation.long_description" type="string">
  A detailed explanation of the remediation, which can include formatting such as markdown for better readability.
</ResponseField>
<ResponseField name="remediation.short_remediation" type="string">
  A concise summary of the steps to remediate the finding.
</ResponseField>
<ResponseField name="remediation.long_remediation" type="string">
  A detailed, step-by-step guide to remediate the finding, which can include formatting such as markdown for better readability.
</ResponseField>
