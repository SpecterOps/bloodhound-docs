OpenGraph data always starts with the same JSON payload structure for nodes and edges. When you also install an extension definition schema, BloodHound can treat that data as a structured graph for pathfinding, findings, and other analysis.

<OpengraphLibrary />

## How OpenGraph works

OpenGraph gives you two ways to bring non-default data into BloodHound:

- Use a valid data payload to ingest and explore OpenGraph data as a generic graph.
- Install an extension definition schema when you want BloodHound to understand relationship types, traversability, environments, and findings for structured analysis.

Use the following guidance to decide which path fits your goal:

<OpenGraphSchemaPaths />

If you need the payload requirements only, start with [OpenGraph Data](/opengraph/developer/data-overview). If you need structured graph behavior, continue to [Extension Definition Schema](/opengraph/developer/extension-definition-schema).

## What extensions add

Extensions let you move from raw graph ingestion to OpenGraph data that BloodHound can analyze more deeply. Depending on the project, an extension can define:

- Node and relationship kinds for your data model
- Traversability rules used for pathfinding and analysis
- Environments and relationship findings for structured output
- Supporting collector, saved query, and Privilege Zone content

Read [Graph Structure](/opengraph/extensions/manage) for the deeper explanation of generic versus structured graphs, extension management workflow, and feature differences.
