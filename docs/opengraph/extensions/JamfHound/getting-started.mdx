This guide walks you through setting up JamfHound end-to-end: creating an Auditor account in JAMF Pro, running the collector, and uploading the results.

## Register the JamfHound Extension in BloodHound

Register the JamfHound extension by uploading the extension files using the built-in upload feature in BloodHound:

- `bhce-jamfhound-extension.json` — jamfHound Extension

## Register JamfHound Node Icons

Register the JAMF Pro node icons by uploading the `jamf-custom-nodes.json` file from the distribution bundle
using the [BloodHound API](/opengraph/custom-icons).

## Import Custom Queries

Import the [JamfHound custom queries](reference/queries) by uploading the `custom-queries/*.json` files to BloodHound on the Explore page.

## Run the JamfHound Collector

Run the JamfHound collector against your JAMF Pro tenant using the credentials from the auditor account.

<Card title="Running the Collector" icon="person-running" href="/opengraph/extensions/JamfHound/running-the-collector">
  Run the JamfHound collector to gather data from your JAMF Pro tenant.
</Card>

## Upload Collected Data to BloodHound

Upload the generated Collection_xx_xx/JAMFcollection.json file from the output directory to BloodHound.

<Note>Okta nodes, edges, and integrations can be added to the collection by using appending the --okta argument during collection.</Note>
