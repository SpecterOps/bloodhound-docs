This guide walks you through setting up OktaHound end-to-end: creating an API service application in Okta,
registering the extension in BloodHound, running the collector, and uploading the results.

## Register the OktaHound Extension in BloodHound

Register the Okta extension (and any other relevant extensions from the bundle) by uploading the extension files using the built-in upload feature in BloodHound:

- `bhce-okta-extension.json` — Okta Extension
- `bh-scim-extension.json` — SCIM Extension
- `bhce-github-extension.json` — GitHub Extension
- `bhce-jamfhound-extension.json` — jamfHound Extension
- `bhce-1passhound-extension.json` — 1PassHound Extension

<Info>You only need to register the extensions that are relevant to your environment. The Okta extension is required; the others are optional.</Info>

## Register OktaHound Node Icons

Register the Okta node icons by uploading the `okta-custom-nodes.json` file from the distribution bundle
using the [BloodHound API](/opengraph/custom-icons).

## Import Custom Queries

Import the [Okta custom queries](reference/queries) by uploading the `custom-queries/*.json` files to BloodHound on the Explore page.

## Run the OktaHound Collector

Run the OktaHound collector against your Okta organization using the credentials from the API service application.

<Card title="Running the Collector" icon="person-running" href="/opengraph/extensions/OktaHound/running-the-collector">
  Run the OktaHound collector to gather data from your Okta organization.
</Card>

## Upload Collected Data to BloodHound

Upload the generated JSON files from the output directory to BloodHound in this order:

1. `okta-graph.json`
2. `okta-graph-ad.json` (optional — includes Active Directory integration data)
3. `okta-graph-hybrid.json` (optional — includes hybrid AD/Okta attack path data)

<Note>The AD-related files are only generated when OktaHound detects Active Directory integration in your Okta environment.</Note>
