## Prerequisites

Before you begin, [ensure the BloodHound graph database is PostgreSQL](/opengraph/requirements#version-requirements). For best performance, BloodHound requires PostgreSQL as the graph database rather than Neo4j.

## Install the Extension

The GitHub extension includes a schema that tells BloodHound how to model and analyze data from your GitHub organization. You must install the extension before you upload data generated by a collector.

- **BloodHound Community**: Download the latest [GitHub extension definition schema file](https://github.com/SpecterOps/openhound-github/blob/main/extension/schema.json) from the OpenHound GitHub repository and install it on the [OpenGraph Management](/opengraph/extensions/manage) page in BloodHound.

- **BloodHound Enterprise**: Contact your account team to get the enterprise version of the schema installed, including BloodHound Enterprise findings.

After the extension is installed, BloodHound can ingest and analyze GitHub data.

### Optional Schemas

If your GitHub environment uses SCIM, upload the [bh-scim-extension.json](https://github.com/SpecterOps/bloodhound-scim-extension/blob/main/bh-scim-extension.json) schema as well. This schema provides a shared model for provisioned users and groups across cloud identity providers and applications.

If GitHub is connected to other BloodHound-supported data sources in your environment, such as Okta, upload the corresponding schema too, or contact your Enterprise account team. Doing so ensures those cross-platform relationships are modeled correctly in BloodHound.

## Import Cypher Queries

The GitHub extension includes [Cypher queries](/opengraph/extensions/github/queries) to help identify attack paths and misconfigurations in GitHub environments. Import the GitHub query JSON files into BloodHound using the [query import workflow](/analyze-data/explore/cypher-search#import-and-export).

You can then run the imported queries on the **Explore** page.

<Note>Cypher queries that reference node or edge kinds not present in the database will fail without the extension schema.</Note>

## Collect and Upload GitHub Data

There are currently two ways to collect GitHub data for this extension:

- [OpenHound GitHub collector](/openhound/collectors/github/overview): The SpecterOps-supported collector for GitHub. Use this if you want the current documented and supported collection path.
- [GitHound collector](https://github.com/SpecterOps/GitHound): An alternative collector that can also produce data for the GitHub extension.

<Tip>Prefer the supported [OpenHound GitHub collector](/openhound/collectors/github/overview).</Tip>

## Configure Privilege Zones

Use the [Privilege Zone rules](/opengraph/extensions/github/privilege-zone-rules) to classify high-value GitHub assets.

The extension also includes specialized (Privilege Zone) queries for [creating Cypher-based Privilege Zone rules](/analyze-data/privilege-zones/rules#cypher).

Read the [Tier Zero Classification](/opengraph/extensions/github/tier-zero) page to understand the rationale behind the Tier Zero Privilege Zone rules.
