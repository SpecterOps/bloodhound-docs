When OpenGraph was introduced in BloodHound v8.0.0, it supported a generic data ingestion model that enabled rapid iteration and flexibility for early OpenGraph projects. This model required data payloads to conform to basic node, edge, and metadata schemas only.

To enable enhanced features and improve consistency across OpenGraph projects, BloodHound now supports an extension-based data ingestion model. This model allows you to register and manage the structures that shape your OpenGraph data in BloodHound. It requires data payloads to conform to an extension schema.

<Note>BloodHound still supports generic data ingestion. For new OpenGraph projects and updates to existing projects, use extension-based ingestion to take advantage of enhanced platform behavior and support.</Note>

## Key concepts

OpenGraph extensions are defined by a schema that specifies the structure and behavior of OpenGraph data for a specific identity provider, cloud service, or other platform. Review the following key concepts to understand how extension-based data ingestion works in BloodHound:

| Concept | Description |
|---|---|
| **Data payload** | The extension-based or generic data generated by an OpenGraph collector that you upload to BloodHound. |
| **Extension-based data** | Data payloads that conform to an extension schema, enabling enhanced features and support in BloodHound. |
| **Extension schema** | A schema that defines the structure and behavior of OpenGraph data, including source, custom node and edge definitions, environment identification, and findings. |
| **Generic data** | Data payloads that conform to basic OpenGraph node, edge, and metadata schemas only. |
| **Collector** | A tool that authenticates to a third-party platform and generates a data payload that BloodHound can ingest. |

## Supported extensions

The following SpecterOps-authored extensions and collectors enable enhanced data ingestion and analysis for specific platforms:

- **GitHound**: Ingest data from GitHub and visualize it in BloodHound alongside your other data sources.
- **JamfHound**: Ingest data from Jamf Pro and visualize it in BloodHound alongside your other data sources.
- **OktaHound**: Ingest data from Okta and visualize it in BloodHound alongside your other data sources.

<Note>These extensions are available in both Community and Enterprise editions of BloodHound, with some features limited to Enterprise.</Note>

## Workflow

Extension-based data ingestion involves the following workflow:

```mermaid
flowchart LR
	subgraph prepare[Prepare]
		a(Download extension and collector)
        a-->b(Install extension)

        c-->d(Generate data payload)

	subgraph upload[Upload]
		f(Data payload)
        f-->g["Import saved queries (optional)"]
		g-->h["Create Privilege Zone rules (optional)"]
	end

	subgraph use[Explore]
		i(Explore data in BloodHound)
	end
    prepare-->upload-->use
```

## Before you begin

For non-built-in platforms, complete the following checks before uploading an extension bundle:

<Steps>
	<Step title="Get extension artifacts">
		Obtain the extension bundle and collector from the [OpenGraph Library](/opengraph/library) or your internal delivery workflow.
	</Step>
	<Step title="Review required permissions">
		Review collector permissions in the extension-specific setup documentation.
	</Step>
	<Step title="Confirm role access">
		Confirm your role includes extension management and data upload permissions.
	</Step>
	<Step title="Validate version compatibility">
		Verify the collector version is compatible with the extension bundle version.
	</Step>
</Steps>

### Update an extension

Collectors and extension schemas are versioned separately to allow for more flexible updates, but this requires coordination to maintain compatibility and support. Follow these guidelines for managing updates:

- Do not update collectors independently without confirming extension schema compatibility.
- Update collectors and extension schemas together whenever possible.
- For Enterprise, coordinate update cycles with your Technical Account Manager.

## Use extension data

After an admin uploads an extension, users can upload compatible data payloads and take advantage of enhanced features in BloodHound. Follow these steps to use extension-based data:

<Steps>
	<Step title="Upload data payloads">
		Upload extension data payloads using the existing file upload flow.
	</Step>
	<Step title="Select extension environment">
		In **Explore**, **Attack Paths**, and **Posture**, select the extension environment when available.
	</Step>
	<Step title="Import saved queries">
		Import extension saved queries separately for MVP.
	</Step>
</Steps>
