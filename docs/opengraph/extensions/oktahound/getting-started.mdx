This guide provides detailed instructions for setting up OktaHound and collecting data from your Okta organization.

1. Download the OktaHound extension and collector
1. Register the OktaHound extension in BloodHound
1. Import the OktaHound custom Cypher queries into BloodHound
1. Create an API service application in Okta
1. Run the OktaHound collector to gather data from your Okta organization
1. Upload the collected data to BloodHound

## Download the Extension and Collector

The OktaHound extension and collector are available for both BloodHound Community and Enterprise editions, however some features are limited to Enterprise only.

- **BloodHound Community** users can download the latest release of OktaHound from the [releases page](https://github.com/SpecterOps/OktaHound/releases) on GitHub.

- **BloodHound Enterprise** customers must contact their Technical Account Manager to obtain the latest release of OktaHound.

## Register the Extension

The OktaHound extension includes a schema that defines how BloodHound structures and analyzes data from your Okta organization. You must register the extension before uploading data generated by the OktaHound collector.

To register the extension in BloodHound, upload the `bhce-okta-extension.json` file on the **OpenGraph Management** page.

OktaHound also includes several optional schemas that recognize additional data sources. You can register these optional schemas if the corresponding data sources are relevant to your environment:

- **SCIM** - `bh-scim-extension.json`
- **GitHub** - `bhce-github-extension.json`
- **Jamf** - `bhce-jamfhound-extension.json`
- **1Password** - `bhce-1passhound-extension.json`
- **Snowflake** - `bhce-snowflake-extension.json`

## Import Cypher Queries

OktaHound provides [custom Cypher queries](/opengraph/extensions/oktahound/reference/queries) to help you identify attack paths and misconfigurations in your Okta environment.

To use these queries, you must first [import](/analyze-data/explore/cypher-search#import-and-export) the `custom-queries/*.json` files into BloodHound. You can then run the queries on the **Explore** page.

## Create an API Service Application

The OktaHound collector uses the Okta API to collect data about your Okta organization. To authenticate to the Okta API, you must create an API service application in Okta and grant it the necessary permissions to access the API.

<Note>Before proceeding to the next section of this guide, you must [create an API service application](/opengraph/extensions/oktahound/okta-app-registration) in Okta.</Note>

## Run the Collector

The OktaHound collector connects to your Okta organization using the API credentials from the service application you created. It collects data about users, groups, applications, roles, and other entities in your Okta environment and generates JSON files that can be uploaded to BloodHound.

<Note>Before proceeding to the next section of this guide, you must [run](/opengraph/extensions/oktahound/collect-data) the OktaHound collector against your Okta organization.</Note>

## Upload Data to BloodHound

The OktaHound collector generates one or more JSON files in the output directory. Each file contains a different subset of data about your Okta environment, such as general configuration data, Active Directory integration data, and hybrid attack path data.

You must upload the generated JSON files to BloodHound in the following order:

1. `okta-graph.json`
1. `okta-graph-ad.json` (optional — includes Active Directory [AD] integration data)
1. `okta-graph-hybrid.json` (optional — includes hybrid AD/Okta attack path data)

<Note>The AD-related files are only generated when OktaHound detects Active Directory integration in your Okta environment.</Note>
