## Prerequisites

Before you begin, [ensure the BloodHound graph database is PostgreSQL](/opengraph/requirements#version-requirements). For best performance, BloodHound requires PostgreSQL as the graph database rather than Neo4j.

## Install the Extension

The Okta extension includes a schema that tells BloodHound how to model and analyze data from your Okta organization. You must install the extension before you upload data generated by a collector.

- **BloodHound Community**: Download the latest [Okta extension definition schema file](https://okta.com/SpecterOps/openhound-okta/blob/main/extension/schema.json) from the OpenHound Okta repository and install it on the [OpenGraph Management](/opengraph/extensions/manage) page in BloodHound.

- **BloodHound Enterprise**: Contact your account team to get the enterprise version of the schema installed, including BloodHound Enterprise findings.

After the extension is installed, BloodHound can ingest and analyze Okta data.

### Optional Schemas

If your Okta environment uses SCIM, upload the [bh-scim-extension.json](https://okta.com/SpecterOps/bloodhound-scim-extension/blob/main/bh-scim-extension.json) schema as well. This schema provides a shared model for provisioned users and groups across cloud identity providers and applications.

If Okta is connected to other BloodHound-supported data sources in your environment, such as Okta, upload the corresponding schema too, or contact your Enterprise account team. Doing so ensures those cross-platform relationships are modeled correctly in BloodHound.

## Import Cypher Queries

The Okta extension includes [Cypher queries](/opengraph/extensions/okta/queries) to help identify attack paths and misconfigurations in Okta environments. Import the Okta query JSON files into BloodHound using the [query import workflow](/analyze-data/explore/cypher-search#import-and-export).

You can then run the imported queries on the **Explore** page.

<Note>Cypher queries that reference node or edge kinds not present in the database will fail without the extension schema.</Note>

## Collect and Upload Okta Data

There are currently two ways to collect Okta data for this extension:

- [OpenHound Okta collector](/openhound/collectors/okta/overview): The SpecterOps-supported collector for Okta. Use this if you want the current documented and supported collection path.
- [OktaHound collector](https://okta.com/SpecterOps/OktaHound): An alternative collector that can also produce data for the Okta extension.

<Tip>Prefer the supported [OpenHound Okta collector](/openhound/collectors/okta/overview).</Tip>

## Configure Privilege Zones

Use the [Privilege Zone rules](/opengraph/extensions/okta/privilege-zone-rules) to classify high-value Okta assets.

The extension also includes specialized (Privilege Zone) queries for [creating Cypher-based Privilege Zone rules](/analyze-data/privilege-zones/rules#cypher).
