The BloodHound extension feature is currently available in preview exclusively for BloodHound Enterprise customers. To get started, contact your Technical Account Manager to obtain the latest Enterprise release of GitHound.

## Prerequisites

Before you begin, [configure and run](/opengraph/extensions/githound/collect-data) the GitHound collector against your GitHub organization to generate a data payload for BloodHound. GitHound supports collection via [Personal Access Token](/opengraph/extensions/githound/configure-pat) or [GitHub App Installation](/opengraph/extensions/githound/configure-app).

## Register the Extension

The GitHound extension includes a schema that tells BloodHound how to model and analyze data from GitHub. You must register the extension before you upload data generated by the GitHound collector.

On the **OpenGraph Management** page, upload the schema file.

## Upload Data to BloodHound

Upload the generated `githound_<orgId>.json` file from the output directory to BloodHound.

If you do not have a GitHub Enterprise environment or want to test GitHound before collecting from your own environment, sample data sets are included in the `./samples/` directory of the [GitHound repository](https://github.com/SpecterOps/GitHound).

## Import Cypher Queries

GitHound provides [custom Cypher queries](/opengraph/extensions/githound/reference/queries) to help you identify attack paths and misconfigurations in your GitHub organization. These queries are included in the `saved-queries` directory of the GitHound extension.

To use these queries, you must first [import](/analyze-data/explore/cypher-search#import-and-export) the `saved-queries/*.json` files into BloodHound. You can then run the queries on the **Explore** page.

<Tip>GitHound also provides [specialized queries](/opengraph/extensions/githound/reference/privilege-zone-rules) in a separate `pz-rules` directory for creating or updating [Cypher-based](/analyze-data/privilege-zones/rules#cypher) Privilege Zone rules.</Tip>
