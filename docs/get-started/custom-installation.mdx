The recommended installation method for most users is the [BloodHound CLI](/get-started/quickstart/community-edition-quickstart). However, if you need more control over the installation process or want to customize specific aspects of your BHCE environment, this guide provides alternative installation methods and customization options.

Use cases for custom installation include:

- Switch between PostgreSQL and Neo4j backends
- Run multiple BHCE instances on a single machine

<Prerequisites />

## Install with Docker Compose

This installation method provides more control over each configuration file and works well for running multiple BHCE instances on a single machine. See the [Customizations](#customizations) section below for details.

1. Create a directory to store the configuration files for your custom BHCE installation.

1. Download the <a href="https://github.com/SpecterOps/BloodHound/blob/main/examples/docker-compose/docker-compose.yml" download><code>docker-compose.yml</code></a> and <a href="https://github.com/SpecterOps/BloodHound/blob/main/examples/docker-compose/bloodhound.config.json" download><code>bloodhound.config.json</code></a> files and copy them into the directory you created.

## Build from source

You can also build the BHCE code from source if you want to contribute to the project or need to customize the application beyond what is possible with configuration files.

### Prerequisite

The following table lists the minimum requirements to build BloodHound CE from source:

<Note>These requirements are higher than the minimum specifications needed to run BHCE.</Note>

| Requirement | Version/Specification |
| --- | --- |
| RAM | 16GB |
| Processor cores | 8 |
| Go | 1.24 |
| Node.js | 22 |
| Yarn | 3.6 |
| Runtime | Docker Desktop |

### Set up your environment

The code repository contains all the necessary files to build BloodHound CE. Follow these steps to set up your development environment:

1. Clone the BloodHound repository from GitHub:

    ```bash
    git clone https://github.com/SpecterOps/BloodHound.git
    ```

1. Start Docker Desktop and ensure it is running.

1. Navigate to the cloned repository directory:

    ```bash
    cd BloodHound
    ```

1. Install the required Go dependencies:

    ```bash
    just init
    ```

1. Start the development environment:

    ```bash
    just bh-dev
    ```

See the source code [wiki](https://github.com/SpecterOps/BloodHound/wiki/Development#quick-start) on GitHub for more details.

### Compile BHCE

The BloodHound team maintains a Python tool called [`stbernard`](https://github.com/SpecterOps/BloodHound/wiki/packages/go/stbernard/README.md) for building and testing the project. To build locally, run `just build`. The build process generates all artifacts in the `dist/` directory.

See the source code [wiki](https://github.com/SpecterOps/BloodHound/wiki/Development#building) on GitHub for more details.

## Customizations

This section describes common customizations you can make to your BHCE installation.

### Change backend database

PostgreSQL provides significant advantages over Neo4j as a backend, particularly in terms of speed. However, it has some drawbacks, the most notable being limited support for certain `Cypher` verbs and syntax. For the latest information on supported Cypher syntax in PostgreSQL, see [Supported Cypher Syntax](/analyze-data/cypher-supported).

#### PostgreSQL

If you are currently using a Neo4j backend database and want to change to PostgreSQL, follow these steps.

1. Edit the `bloodhound.config.json` file and add a line in the main section to use PostgreSQL as the graph driver:

    ```json
    ...
    "graph_driver": "pg",
    ...
    ```

    This is an example of adding the line between `default_password` and `log_level`, but it can be anywhere at that "level".

    ```json
    {
      "bind_addr": "0.0.0.0:8080",
      "collectors_base_path": "/etc/bloodhound/collectors",
      "default_admin": {
        "password": "<SOMETHING>",
        "principal_name": "admin"
      },
      "default_password": "<SOMETHING>",
      "graph_driver": "pg", // [!code ++]
      "log_level": "INFO",
      "log_path": "bloodhound.log",
      "metrics_port": ":2112",
      "recreatedefaultadmin": "false",
      "root_url": "http://127.0.0.1:8080",
      "tls": {
        "cert_file": "",
        "key_file": ""
      },
      "version": 1,
      "work_dir": "/opt/bloodhound/work"
    }                        
    ```

1. Clean up the `docker-compose.yml` file.

    - Delete the whole section about `graph_db`

      ```yaml
        graph-db: # [!code --:23]
          labels:
            name: "bhce_neo4j"
          image: docker.io/library/neo4j:4.4
          environment:
            - NEO4J_AUTH=${NEO4J_USER:-neo4j}/${NEO4J_SECRET:-bloodhoundcommunityedition}
            - NEO4J_dbms_allow__upgrade=${NEO4J_ALLOW_UPGRADE:-true}
          # Database ports are disabled by default. Please change your database password to something secure before uncommenting
          ports:
            - 127.0.0.1:${NEO4J_DB_PORT:-7687}:7687
            - 0.0.0.0:${NEO4J_WEB_PORT:-7474}:7474
          volumes:
            - ${NEO4J_DATA_MOUNT:-neo4j-data}:/data
          healthcheck:
            test:
              [
                "CMD-SHELL",
                "wget -O /dev/null -q http://localhost:7474 || exit 1"
              ]
            interval: 10s
            timeout: 5s
            retries: 5
            start_period: 30s
      ```

    - Delete the 2 lines at the end of the `bloodhound` section:

      ```yaml
            graph-db: # [!code --:2]
              condition: service_healthy
      ```

    - Delete the following line from the `volumes` section:

      ```yaml
        neo4j-data: # [!code --]
      ```

### Run multiple instances simultaneously

You might want to run multiple instances to:

- Test a Neo4j backend alongside a PostgreSQL backend
- Evaluate a newer version without affecting your current installation
- Isolate data for different engagements

<Note>You must bind each instance to a different port on your machine. The steps below show you which settings to edit.</Note>

1. Change the web portal port in `docker-compose.yml`

  Update the BloodHound service port binding to use a different port. In this example, we'll use port `8585` instead of the default `8080`:

  ```yaml
      - ${BLOODHOUND_HOST:-127.0.0.1}:${BLOODHOUND_PORT:-8585}:8080
  ```

1. If you are running multiple Neo4j databases, update the Neo4j Web Console port in the `docker-compose.yml` file. Change the port binding and health check to use a different port. In this example, we use port `7575` instead of the default `7474`:

  ```yaml
  - 0.0.0.0:${NEO4J_WEB_PORT:-7575}:7474
  ```

  Also update the health check to use the new port:

  ```yaml
  "wget -O /dev/null -q http://localhost:7575 || exit 1"
  ```

  <Note>The Neo4j database port `7687` runs only inside the Docker container, so you do not need to change it.</Note>
