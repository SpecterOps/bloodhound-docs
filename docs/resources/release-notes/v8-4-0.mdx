## Announcements

### BloodHound Enterprise Splunk App (v1.1.0)

We are excited to announce the v1.1.0 release of the [BloodHound Enterprise Splunk App](https://splunkbase.splunk.com/app/7818). The app allows you to visualize and analyze BloodHound data directly within Splunk, providing enhanced capabilities for security analysis and threat hunting. Notable features in this release include:

- New **Posture History** dashboard showing key security metrics and trends over time
- Support for proxy configurations

## Summary

- **BloodHound**

  - New and Improved Features

    - Privilege Zones
      - {/*BED-6516*/} Renamed "Selectors" to "Rules" in the UI to improve understanding.
      - {/*BED-6698*/} Quick-add rules now default to certifying **Initial Members** (instead of **Off**).
      - {/*BED-6704*/} Added description text for **Enable Analysis** and polished details panel field naming.
      - {/*BED-6624*/} Dynamic **Edit** button now reflects editing context (Zone, Label, or Rule).
      - {/*BED-6625*/} Refined selection UI to clearly highlight only the currently selected item.
      - {/*BED-6630*/} Added icons to the **Object Count** list.
      - {/*BED-6709*/} Added glyph values (gem, skull) via migration for default **Tier Zero** and **Owned** labels.
      - {/*BED-6158*/} Replaced references to "tier" with "zone".

    - Findings and Analysis
      - {/*BED-6744*/} _[BHE only]_ Rewrote `T0MarkSensitive` finding language to more accurately reflect risk and removed non-Tier Zero wording.
      - {/*BED-6697*/} Limited `T0MarkSensitive` processing to Tier Zero zones only.
      - {/*BED-6256*/} _[BHE only]_ Removed unnecessary Tier Zero verbiage from remediation long descriptions.

    - Roles and Permissions
      - {/*BED-6674*/} Added a new **Auditor** role to provide users with read-only access to all application settings and configurations, including audit log viewing capabilities.

    - General Improvements
      - {/*BED-6629*/} Renamed `SYSTEM` identifier to `BloodHound`.

  - Bug Fixes

    - Privilege Zones
      - {/*BED-6711*/} Improved clarity of error messaging when users attempt to add a rule that already exists.
      - {/*BED-6659*/} _[BHE only]_ Removed outdated Tier Zero/non-Tier Zero labels for non-Tier Zero zones in **Findings** panel titles.

    - Analysis and Data Quality
      - {/*BED-6418*/} Updated tooltips for **Local Group** and **Session Completeness** metrics on the **Data Quality** and **Posture** pages to reflect that BloodHound considers any enabled computer that has logged into the domain in the last 14 days to be active.
      - {/*BED-5021*/} Ensured `CanRDP` edges are properly created when Citrix RDP support is enabled and resolved stale edges behavior.

    - UI and Display
      - {/*BED-6705*/} _[BHE only]_ Resolved missing graph icons when opening **Explore** page deep links in a new tab.
      - {/*BED-6545*/} _[BHE only]_ Updated the messaging in the **Client Token Info** modal to provide relevant guidance based on collector client type (AzureHound or SharpHound).

    - API and Backend
      - {/*BED-6664*/} Corrected inaccurate description of the `asset_group_tag_id` query parameter for the `GET /api/v2/posture-history/{data_type}` API operation.
      - {/*BED-5574*/} Fixed an issue preventing users with the Admin role from deleting users who have uploaded data on the **File Ingest** page.
