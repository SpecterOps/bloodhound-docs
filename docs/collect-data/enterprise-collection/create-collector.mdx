## Purpose

This guide explains how to create a BloodHound Enterprise collector client. BloodHound Enterprise supports two types of collector clients:

- **SharpHound Enterprise** - Collects data from Active Directory environments
- **AzureHound Enterprise** - Collects data from Entra ID environments

It should be used by Administrators when deploying SharpHound Enterprise or AzureHound Enterprise.

## Prerequisites

- Having deployed a BloodHound Enterprise Tenant
- Logged in as a user assigned a [role](/manage-bloodhound/auth/users-and-roles) authorized to create a collector client

## Process

The process to create a BloodHound Enterprise collector client consists of the following steps:

<Steps>
  <Step title="Open the Manage Clients page">
  In the left menu, click **Administration** > **Manage Clients**.
  </Step>

  <Step title="Create a client">
    1. On the right side of the page, click **Create Client**.

    1. Select a client from the drop-down.

        Options include:

        - **SharpHound Enterprise Client** for Active Directory data collection
        - **AzureHound Enterprise Client** for Entra ID data collection

          <Tabs>
            <Tab title="SharpHound Enterprise Client">
              Enter details for the SharpHound Enterprise Client and click **Create**.

              | Field | Required | Description |
              |-------|----------|-------------|
              | **Client Name** | Required | A descriptive name for the collector client |
              | **Authentication Method** | Required | Select the authentication method:<br/>- **API Authentication**: Traditional token-based authentication (default)<br/>- **Integrated Windows Authentication (IWA)**: Windows-based authentication via ADFS |
              | **Collection Schedule** | Optional | |
              | **Advanced Options** | Optional | |

              #### API Authentication
              For API Authentication clients, no additional fields are required beyond the client name.

              #### Integrated Windows Authentication (IWA)
              For IWA clients, you will need to provide ADFS configuration details:

              | Field | Required | Description |
              |-------|----------|-------------|
              | **Issuer ID (Well-known)** | Required | The ADFS well-known endpoint URL, typically: `https://adfs.example.com/.well-known/openid-configuration` |
              | **Issuer Override ID** | Optional | An optional override for the token issuer address if your ADFS configuration uses a different issuer URL for token validation |

              <Frame>
                <img
                  src="/images/data_collectors/create-sharphound.png"
                  alt="A view of the Create Client screen for SharpHound Enterprise"
                />
              </Frame>
            </Tab>

            <Tab title="AzureHound Enterprise Client">
              Enter details for the AzureHound Enterprise Client and click **Create**.

              | Field | Required | Description |
              |-------|----------|-------------|
              | **Client Name** | Required | A descriptive name for the collector client |
              | **Collection Schedule** | Optional | |

              <Note>AzureHound clients always use API Authentication.</Note>

                <Frame>
                  <img
                    src="/images/data_collectors/create-azurehound.png"
                    alt="A view of the Create Client screen for AzureHound Enterprise"
                  />
                </Frame>
            </Tab>
          </Tabs>
  </Step>
  
  <Step title="Save the client token or configuration">
  A pop-up window will appear with client configuration information; follow the instructions in it before clicking **CLOSE**.

    <Tabs>
      <Tab title="API Authentication">
        A _Client Token Info_ window will appear with authentication credentials. Copy and save the token information before closing.

        <Note>The token information is required to [configure](/install-data-collector/install-sharphound/local-configuration) the collector client application in the `auth.json` file.</Note>

          <Frame>
            <img
              src="/images/data_collectors/sharphound-client-token-info.png"
              alt="A view of the client token info screen for SharpHound Enterprise"
            />
          </Frame>
        </Tab>

      <Tab title="Integrated Windows Authentication (IWA)">
        A _Client Configuration Info_ window will appear with the Client ID and other configuration details needed to set up ADFS.

        <Note>The Client ID and configuration details are required to [configure ADFS](/install-data-collector/install-sharphound/configure-adfs-iwa) and to [configure](/install-data-collector/install-sharphound/local-configuration) the collector client application in `settings.json`.</Note>

          <Frame>
            <img
              src="/images/data_collectors/sharphound-client-config-info.png"
              alt="A view of the client configuration info screen for SharpHound Enterprise IWA"
            />
          </Frame>
        </Tab>

      <Tab title="AzureHound Enterprise Client">
        A _Client Token Info_ window will appear with authentication credentials. Copy and save the token information before closing.

        <Note>The token information is required to [configure](/install-data-collector/install-azurehound/create-configuration) the collector client application.</Note>

          <Frame>
            <img
              src="/images/data_collectors/azurehound-client-token-info.png"
              alt="A view of the client token info screen for AzureHound Enterprise"
            />
          </Frame>
        </Tab>
    </Tabs>
  </Step>
</Steps>

## Outcome

The collector client will appear in the clients table with a **Status** of **Unconfigured**. The table displays each client's **Authentication Method**, showing either "API Authentication" or "Integrated Windows Authentication" based on the method selected during client creation.

### Next Steps

- **For API Authentication clients**: Use the token information to [configure](/install-data-collector/install-sharphound/local-configuration) the collector client application in the `auth.json` file.
- **For IWA clients**: Follow the [ADFS configuration guide](/install-data-collector/install-sharphound/configure-adfs-iwa) to set up ADFS, then [configure](/install-data-collector/install-sharphound/local-configuration) the collector client application in `settings.json`.

See [Install SharpHound Enterprise](/install-data-collector/install-sharphound) or [Install AzureHound Enterprise](/install-data-collector/install-azurehound) for more information.
