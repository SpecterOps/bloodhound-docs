BloodHound Enterprise supports [API key expiration](/manage-bloodhound/auth/api-key-expiration) to help you align with internal security and compliance requirements. API key expiration controls how long API tokens remain valid.

Expiration applies equally to all three API token types:

- **Personal API tokens and Token IDs** — created by individual users for API access.
- **Non-personal API tokens and Token IDs** — created for integrations such as Splunk, ServiceNow, and Cortex XSOAR.
- **Collector API tokens and Token IDs** — created for SharpHound Enterprise and AzureHound Enterprise data collectors.

| Setting or action | Behavior |
| --- | --- |
| Default validity period | New tokens expire after 90 days by default. |
| Custom validity period | Administrators can set token validity from 1 to 365 days. |
| Feature turned on | Existing tokens receive expiration based on the configured value. New tokens follow the same policy. |
| Validity period changed | Each token keeps the earlier of its current expiration and now + configured days. |
| Feature turned off | Expiration enforcement stops, expiration dates are cleared, and tokens no longer expire. |

Use API key expiration to reduce long-lived credentials and support internal compliance requirements. When API key expiration is enabled, plan to rotate API tokens and update integration configurations before they expire to prevent disruptions.

## Before you begin

- You must have the **Administrator** role.
- Plan key rotation for all integrations and collector clients before changing this setting.
- BloodHound does not automatically rotate API keys.

## Configure API key expiration

<Steps>
  <Step title="Open the API key expiration setting">
    1. Go to **Settings** <Icon icon="gear" iconType="solid" />.
    1. Select **Administration**.
    1. Open the API key expiration setting.

    _Screenshot placeholder: API key expiration setting in BloodHound Enterprise administration._
  </Step>

  <Step title="Set expiration policy and save">
    1. Turn the setting on or off.
    1. If enabled, set the validity period in days.
    1. Enter a value from 1 to 365.
    1. Click **Save**.

    _Screenshot placeholder: confirmation modal when saving API key expiration policy changes._
  </Step>
</Steps>

## API key expiration behavior

| Setting or action | Behavior |
| --- | --- |
| Default validity period | New API tokens expire after 90 days. |
| Custom validity period | Administrators can set a value from 1 to 365 days. |
| Feature turned on | Existing API tokens are updated to expire based on the current policy. New tokens follow the same policy. |
| Validity period changed | Each token keeps the earlier of its current expiration and now + configured days. |
| Feature turned off | Expiration checks stop, expiration dates are cleared, and tokens no longer expire. |

<Warning>
Disabling API key expiration removes expiration dates from existing API tokens. Re-enabling expiration later applies a new expiration policy again.
</Warning>

## Rotate keys manually

BloodHound only shows the API token and Token ID when you create them. If you lose either value or a token expires, you must create a replacement and update the dependent integration.

For API key creation steps, see [Work With the BloodHound API](/integrations/bloodhound-api/working-with-api).
