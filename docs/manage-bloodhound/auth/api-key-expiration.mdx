Use API key expiration to reduce long-lived credentials and support internal compliance requirements.

This setting applies to all API key types in BloodHound Enterprise:

- **Personal keys** — created by individual users for API access.
- **Non-personal keys** — created for integrations such as Splunk, ServiceNow, and Cortex XSOAR.
- **Collector keys** — created for SharpHound Enterprise and AzureHound Enterprise data collectors.

## Before you begin

- You must have the **Administrator** role.
- Plan key rotation for all integrations and collector clients before changing this setting.
- BloodHound does not automatically rotate API keys.

## Configure API key expiration

<Steps>
  <Step title="Open the API key expiration setting">
    1. Go to **Settings** <Icon icon="gear" iconType="solid" />.
    1. Select **Administration**.
    1. Open the API key expiration setting.

    _Screenshot placeholder: API key expiration setting in BloodHound Enterprise administration._
  </Step>

  <Step title="Set expiration policy and save">
    1. Turn the setting on or off.
    1. If enabled, set the validity period in days.
    1. Enter a value from 1 to 365.
    1. Click **Save**.

    _Screenshot placeholder: confirmation modal when saving API key expiration policy changes._
  </Step>
</Steps>

## API key expiration behavior

| Setting or action | Behavior |
| --- | --- |
| Default validity period | New API keys expire after 90 days. |
| Custom validity period | Administrators can set a value from 1 to 365 days. |
| Feature turned on | Existing API keys are updated to expire based on the current policy. New keys follow the same policy. |
| Validity period changed | Each key keeps the earlier of its current expiration and now + configured days. |
| Feature turned off | Expiration checks stop, expiration dates are cleared, and keys no longer expire. |

<Warning>
Disabling API key expiration removes expiration dates from existing keys. Re-enabling expiration later applies a new expiration policy again.
</Warning>

## Rotate keys manually

BloodHound only shows a key value when you create it. If you lose a key value or a key expires, you must create a replacement key and update the dependent integration.

For API key creation steps, see [Work With the BloodHound API](/integrations/bloodhound-api/working-with-api).
